28    /**
29    * An abstract subclass of <code>java.io.InputStream</code> that allows seeking
30    * within the input, similar to the <code>RandomAccessFile</code> class.
31    * Additionally, the <code>DataInput</code> interface is supported and extended
32    * to include support for little-endian representations of fundamental data
33    * types.
34    *
35    * <p> In addition to the familiar methods from <code>InputStream</code>, the
36    * methods <code>getFilePointer()</code>, <code>seek()</code>, are defined as in
37    * the <code>RandomAccessFile</code> class. The <code>canSeekBackwards()</code>
38    * method will return <code>true</code> if it is permissible to seek to a
39    * position earlier in the stream than the current value of
40    * <code>getFilePointer()</code>. Some subclasses of
41    * <code>SeekableStream</code> guarantee the ability to seek backwards while
42    * others may not offer this feature in the interest of providing greater
43    * efficiency for those users who do not require it.
44    *
45    * <p> The <code>DataInput</code> interface is supported as well. This included
46    * the <code>skipBytes()</code> and <code>readFully()</code> methods and a
47    * variety of <code>read</code> methods for various data types.
48    *
49    * <p> Three classes are provided for the purpose of adapting a standard
50    * <code>InputStream</code> to the <code>SeekableStream</code> interface.
51    * <code>ForwardSeekableStream</code> does not allows seeking backwards, but is
52    * inexpensive to use. <code>FileCacheSeekableStream</code> maintains a copy of
53    * all of the data read from the input in a temporary file; this file will be
54    * discarded automatically when the <code>FileSeekableStream</code> is
55    * finalized, or when the JVM exits normally.
56    * <code>FileCacheSeekableStream</code> is intended to be reasonably efficient
57    * apart from the unavoidable use of disk space. In circumstances where the
58    * creation of a temporary file is not possible,
59    * <code>MemoryCacheSeekableStream</code> may be used.
60    * <code>MemoryCacheSeekableStream</code> creates a potentially large in-memory
61    * buffer to store the stream data and so should be avoided when possible.
62    *
63    * <p> The <code>FileSeekableStream</code> class wraps a <code>File</code> or
64    * <code>RandomAccessFile</code>. It forwards requests to the real underlying
65    * file. It performs a limited amount of caching in order to avoid excessive
66    * I/O costs.
67    *
68    * <p> The <code>SegmentedSeekableStream</code> class performs a different sort
69    * of function. It creates a <code>SeekableStream</code> from another
70    * <code>SeekableStream</code> by selecting a series of portions or "segments".
71    * Each segment starts at a specified location within the source
72    * <code>SeekableStream</code> and extends for a specified number of bytes. The
73    * <code>StreamSegmentMapper</code> interface and <code>StreamSegment</code>
74    * class may be used to compute the segment positions dynamically.
75    *
76    * <p> A convenience methods, <code>wrapInputStream</code> is provided to
77    * construct a suitable <code>SeekableStream</code> instance whose data is
78    * supplied by a given <code>InputStream</code>. The caller, by means of the
79    * <code>canSeekBackwards</code> parameter, determines whether support for
80    * seeking backwards is required.
81    *
82    */
